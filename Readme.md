Take-home assessment: Junior Full-Stack Engineer
Task: Auth + Simple Buyer Portal (Favourites)
Problem statement
You are building part of the buyer portal for a real-estate broker.
Given a simple skeleton for an API, you must:
1. Implement:
a. User registration and login (email + password).
b. JWT or session-style auth (in-memory or simple DB-backed).
2. Create a simple buyer dashboard page that:
a. Shows the user’s name and role.
b. Lists “My Favourites” (properties) stored against the user.
c. Allows the user to:
i. Add/remove a property to/from favourites.
d. Enforces that users can only see and modify their own favourites.
3. Add basic frontend:
a. A login page and a dashboard page.
b. A simple form or button flow to “like” a property.
4. Implement:
a. Basic server-side validation and error handling.
b. A tiny DB layer (e.g., users, favourites tables or equivalent).
5. Add:
a. A short README with:
i. How to run the app.
ii. Example flows (e.g., sign up → login → add favourite).

Evaluation focus
• Understanding of auth flows and basic security (no raw passwords in DB).
• Simple DB design and CRUD operations.
• Clear separation between frontend and backend.
• ability to read and extend existing code.
• basic UX thinking (e.g., clear success/error messages).
